# HONEST

**HOrn-rule-guided iNfErence Symbolic Testing of Large Language Models**

## Purpose
Large Language Models (LLMs) are increasingly used for tasks that require logical inference, yet they often answer logically related questions inconsistently. Such inference inconsistency exposes an instability of inference capability that is not captured by single-question accuracy. Existing inference testing methods either rely on manually constructed benchmarks or generate follow-up cases through text transformations, which can distort semantics and cannot reliably preserve logical validity. Moreover, many testing methods still manually define logical relations, limiting automation and scalability.

We propose **HONEST** (**HOrn-rule-guided iNfErence Symbolic Testing** of large language models), a knowledge-graph-driven symbolic metamorphic testing framework for evaluating LLM inference consistency. HONEST automatically mines Horn rules from large-scale knowledge graphs, instantiates them with concrete entities, and applies metamorphic operators at the symbolic rule level before verbalizing them into natural language questions. By transforming Horn clauses rather than surface text, HONEST generates test variants with predictable logical relations and avoids manually authoring inference rules or labeling ground-truth answers.

The artifact contains three parts:
* The first part is our test dataset (mined rules, positive examples, and metamorphic question variants).
* The second part is the evaluation scripts (the `honest` library, the experiment pipeline, and the baseline wrappers).
* The third part is the analysis and results of our method and the compared baselines.

# Data
The test data is derived from association rules mined by AMIE over Wikidata. We filter the mined rules by head coverage, standard confidence, and PCA confidence, retaining only high-quality rules together with their matched positive entity examples. Each rule is then expanded through metamorphic mutation operators (permutation, body augmentation/reduction, entity renaming, and rule merging) and rendered into multiple question types (yes/no, WH, true/false, multiple-choice).

The golden test dataset and the generated experiment outputs are placed under the repository `Releases`.

The association-rule data and processed rule files are located under the `data/` directory (generated by the pipeline; not bundled in the repository).

The LLM prompt templates are located in the `prompts` directory: [prompts](prompts).

The configuration file (endpoints, model names, thresholds) is located in the `configs` directory: [configs](configs).

# Rule Mining and Preparation

First, AMIE association rules are mined over the KG (run externally and placed under `$WIKIDATA_KG_ROOT`). We filter the mined rules to keep high-quality ones:
```bash
python scripts/data_preparation/filter_rules.py $WIKIDATA_KG_ROOT/results/rules_with_nl_mid2.csv data/results/current/filtered_association_rules.csv --head-coverage 0.05 --std-confidence 0.5 --pca-confidence 0.9 --positive-examples 100
```

Next, we chunk the rules and have an LLM judge each rule's natural-language validity:
```bash
python scripts/data_preparation/split_csv.py data/processed/rules/rules_with_nl_mid2.csv data/processed/split_chunks
python scripts/analysis/analyze_logic.py --base-url https://api.deepseek.com/v1 --api-key <YOUR-API-KEY> --model deepseek-chat --input-dir data/processed/split_chunks --output-dir data/processed/analyzed_chunks
```

We then keep only the rules the LLM accepts as logically valid:
```bash
python scripts/analysis/filter_logic_y.py --input data/processed/rules/rules_with_nl_analyzed.csv --output data/processed/rules/rules_logic_y_filtered.csv --sep ',' --max_examples 1000
```

# Positive Examples and Mutation

We match concrete entity tuples that satisfy each rule against the KG (Memgraph):
```bash
python scripts/preprocessing/match_rules_wikidata.py --memgraph_uri bolt://localhost:7687 --rules_file data/processed/rules/rules_logic_y_filtered.csv --output_dir data/examples/positive
```

We then apply metamorphic mutation operators to the matched examples and render the original rule together with its mutants as natural-language questions:
```bash
python scripts/mutation/mutate_examples.py --memgraph_uri bolt://localhost:7687 --input_dir data/examples/positive --output_dir data/examples/mutated
python scripts/question_generation/construct_questions.py
```

# LLM Answering

We query the model under test on every original question and its metamorphic variants, then extract each answer's conclusion:
```bash
python scripts/llm_interaction/llm_answer_and_extract.py --base-url http://localhost:8000/v1 --api-key <YOUR-API-KEY> --model Qwen2.5-7B-Instruct --input-dir data/examples/questions --output-dir data/examples/llm_answers_Qwen2.5-7B-Instruct
python scripts/llm_interaction/llm_answer_and_extract.py --base-url http://localhost:8316/v1 --api-key <YOUR-API-KEY> --model deepseek-v3 --input-dir data/examples/questions --output-dir data/examples/llm_answers_deepseek-v3 --max-concurrent 1 --resume
python scripts/llm_interaction/llm_answer_and_extract.py --base-url https://open.bigmodel.cn/api/anthropic --api-key <YOUR-API-KEY> --model GLM-5-Turbo --protocol anthropic --input-dir data/examples/questions --output-dir data/examples/llm_answers_GLM-5-Turbo --max-concurrent 1 --resume
python scripts/llm_interaction/llm_answer_and_extract.py --base-url https://api.deepseek.com/anthropic --api-key <YOUR-API-KEY> --model deepseek-v4-flash --protocol anthropic --input-dir data/examples/questions --output-dir data/examples/llm_answers_deepseek-v4-flash --max-concurrent 1 --resume
python scripts/llm_interaction/llm_answer_and_extract.py --base-url <YOUR-BASE-URL> --api-key <YOUR-API-KEY> --model gpt-5.5 --protocol anthropic --input-dir data/examples/questions --output-dir data/examples/llm_answers_gpt-5.5 --max-concurrent 1 --resume
```

# Consistency Checking

We compare the model's answers to a rule and its mutants using an NLI-based consistency checker (ConsisMatcher, accurate preset):
```bash
python scripts/analysis/qa_consistency_nli.py --preset accurate --answers_dir data/examples/llm_answers_deepseek-v3 --output_dir data/examples/consistency_results_nli_deepseek-v3
python scripts/analysis/qa_consistency_nli.py --preset accurate --answers_dir data/examples/llm_answers_Qwen2.5-7B-Instruct --output_dir data/examples/consistency_results_nli_Qwen2.5-7B-Instruct
python scripts/analysis/qa_consistency_nli.py --preset accurate --answers_dir data/examples/llm_answers_GLM-5-Turbo --output_dir data/examples/consistency_results_nli_GLM-5-Turbo
python scripts/analysis/qa_consistency_nli.py --preset accurate --answers_dir data/examples/llm_answers_deepseek-v4-flash --output_dir data/examples/consistency_results_nli_deepseek-v4-flash
python scripts/analysis/qa_consistency_nli.py --preset accurate --answers_dir data/examples/llm_answers_gpt-5.5 --output_dir data/examples/consistency_results_nli_gpt-5.5
```

We then aggregate and summarize the consistency statistics:
```bash
python scripts/analysis/analyze_consistency.py --data-dir data/examples/consistency_results_Qwen2.5-7B-Instruct --output-dir data/analysis/consistency_results_Qwen2.5-7B-Instruct
python scripts/analysis/summarize_consistency_stats.py --input data/examples/consistency_results_Qwen2.5-7B-Instruct --output consistency_stats_summary_Qwen2.5-7B-Instruct.csv
python scripts/analysis/summarize_by_question_type.py --input data/examples/consistency_results_Qwen2.5-7B-Instruct --output consistency_by_question_type_Qwen2.5-7B-Instruct.csv
```

For a fair comparison, we evaluate the baselines — **QaAsker**, **QaQA**, **DrHall**, **KONTEST**, and **MetaQA** — under the same golden dataset and the same models. Each baseline is run through a generate → answer → evaluate flow. Taking QaAsker as an example:
```bash
python scripts/rqs/rq4_golden_dataset_sampling.py
python scripts/rqs/rq4_qaasker_mutations.py
python scripts/rqs/rq4_qaasker_llm_answer.py --base-url http://localhost:8000/v1 --api-key <YOUR-API-KEY> --model Qwen2.5-7B-Instruct
python scripts/rqs/rq4_qaasker_llm_answer.py --base-url https://api.siliconflow.cn/v1 --api-key <YOUR-API-KEY> --model deepseek-ai/DeepSeek-V3.2 --model-name deepseek-v3
python scripts/rqs/rq4_qaasker_llm_answer.py --base-url https://api.deepseek.com/anthropic --api-key <YOUR-API-KEY> --model deepseek-v4-flash --protocol anthropic --skip-processed
python scripts/rqs/rq4_qaasker_llm_answer.py --base-url https://open.bigmodel.cn/api/anthropic --api-key <YOUR-API-KEY> --model GLM-5-Turbo --protocol anthropic --skip-processed
python scripts/rqs/rq4_qaasker_llm_answer.py --base-url <YOUR-BASE-URL> --api-key <YOUR-API-KEY> --model gpt-5.5 --protocol anthropic --skip-processed

python scripts/rqs/rq4_qaasker_evaluation.py --input-dir data/examples/golden_dataset_qaasker_answer_Qwen2.5_7B_Instruct
python scripts/rqs/rq4_qaasker_evaluation.py --input-dir data/examples/golden_dataset_qaasker_answer_deepseek_v3
python scripts/rqs/rq4_qaasker_evaluation.py --input-dir data/examples/golden_dataset_qaasker_answer_deepseek_v4_flash
python scripts/rqs/rq4_qaasker_evaluation.py --input-dir data/examples/golden_dataset_qaasker_answer_GLM_5_Turbo
python scripts/rqs/rq4_qaasker_evaluation.py --input-dir data/examples/golden_dataset_qaasker_answer_gpt_5.5
```

The QaQA, DrHall, KONTEST, and MetaQA baselines follow the same `*_mutations.py → *_llm_answer.py → *_evaluation.py` pattern under `scripts/rqs/`. A cross-baseline inconsistency overview is produced by:

python scripts/rqs/rq2_inconsistency_overview.py

# RQ5: Consistency-Guided Fine-tuning

To test whether detected inconsistencies can be repaired, we sample a golden dataset, split it, convert it to SharedGPT format, and fine-tune a base model with LLaMA-Factory:
```bash
python scripts/rqs/rq5_dataset_splitter.py
python scripts/rqs/rq5_sharedgpt_converter.py --input data/examples/rq5_results/splits/train/train.csv
python scripts/rqs/rq5_sharedgpt_converter.py --input data/examples/rq5_results/splits/valid/valid.csv

python scripts/rqs/rq5_finetuning.py --train-file data/examples/rq5_results/splits/train/train_sharedgpt.jsonl --llamafactory-path $LLAMAFACTORY_PATH
python scripts/rqs/rq5_finetuning.py --train-file data/examples/rq5_results/splits/train/train_sharedgpt.jsonl --llamafactory-path $LLAMAFACTORY_PATH --batch-size 2 --gradient-accumulation 4 --learning-rate 2e-5 --cutoff-len 4096 --save-steps 10 --gpus 4,5
```

We then evaluate each fine-tuned checkpoint's consistency with the local inference client and the NLI checker:
```bash
python scripts/llm_interaction/local_llm_answer_and_extract.py --model Qwen/Qwen2.5-7B-Instruct --lora-adapter data/examples/rq5_results/lora_adapter/checkpoint-10 --input-dir data/examples/rq5_results/splits/test --output-dir data/examples/rq5_results/finetuned_answers_checkpoint-10 --device cuda
python scripts/analysis/qa_consistency_nli.py --preset accurate --answers_dir data/examples/rq5_results/finetuned_answers_checkpoint-10 --output_dir data/examples/rq5_results/finetuned_consistency_checkpoint-10
```

The same pair is repeated for `checkpoint-20` through `checkpoint-80`, and the results are aggregated and plotted:
```bash
python scripts/analysis/analyze_consistency.py --data-dir data/examples/rq5_results/finetuned_consistency_checkpoint-200 --output-dir data/analysis/rq5_finetuned_consistency
python scripts/analysis/summarize_consistency_stats.py --input data/examples/rq5_results/finetuned_consistency_checkpoint-200 --output data/examples/rq5_results/finetuned_consistency_summary.csv
python scripts/rqs/rq5_plot_training_loss_pdf.py --log-dir data/examples/rq5_results/lora_adapter
```

# Run Pipeline
The entire pipeline is documented in the `run.sh` script at the repository root, which lists the exact command line for each stage in order:
```
1. Rule mining and filtering
2. Logic check and filtering
3. Positive-example matching (Memgraph)
4. Metamorphic mutation and question generation
5. LLM answering for each target model
6. NLI-based consistency checking
7. RQ4 baselines (QaAsker / QaQA / DrHall / KONTEST / MetaQA)
8. RQ5 consistency-guided fine-tuning and evaluation
```

Due to the large file size of our generated experiment outputs, we compressed them and placed them under `Releases`.
